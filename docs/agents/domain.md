# Domain Docs

This repository uses a single-context layout:

- `CONTEXT.md` contains product language, platform boundaries, and the current first slice.
- `docs/adr/` records architecture decisions that should not be re-litigated casually.

Agents should read `CONTEXT.md` before proposing architecture, migrating assets, or implementing GIS+BIM, BIM detail, alert, diagnosis, or work-order behavior.
