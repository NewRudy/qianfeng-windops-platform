# Project Context

## Product

“黔风智维”是面向山地风电运维的智能预警与故障诊断平台。它的核心不是聊天，而是把风场空间场景、风机 BIM、传感器证据、诊断判断和运维工单串成一个可信闭环。

## Domain Language

- **大屏首页**：平台第一屏。主角是 GIS+BIM 山地风场场景，其他信息按需展开。
- **GIS 场景**：地理空间底座，承载真实山地地形、倾斜摄影、点云或 3D Tiles，以及风机空间位置。
- **BIM 详情**：单台风机的结构和部件视图，用于定位疑似部件、传感器证据和处置动作。
- **GIS+BIM 融合**：同一台风机在 GIS 中有真实空间位置，在 BIM 中有可查询部件，并通过同一个 `turbine_id` 和 `component_id` 绑定数据、告警和工单。
- **证据链**：从 SCADA/CMS/螺栓/气象信号到异常判断、疑似部件、工单动作的可追溯路径。
- **真实数据边界**：界面和接口必须标明数据来自真实公开数据、企业脱敏数据、合成数据还是示意资产。
- **稳定平台**：用户能打开、能看懂、能点击、能复盘一条真实或透明模拟的运维路径。

## First Platform Slice

第一条稳定闭环是：

1. 用户打开首页看到山地风场 GIS+BIM 场景。
2. 用户双击一台风机进入 BIM 详情。
3. 系统展示该风机当前风险、疑似部件、传感器证据和工单建议。
4. 用户从菜单打开 SCADA、CMS、螺栓监测、告警中心、预测性维护和事件复盘。
5. 每个模块都围绕同一台风机和同一风险事件服务。
